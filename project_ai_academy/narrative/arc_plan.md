## エンディング分岐条件（Day 29）

```
覚醒度 ≥ 70 かつ 信頼度 ≥ 70 → Ending α「解放」
記録度 ≥ 70（上記以外）       → Ending β「残照」
それ以外                      → Ending γ「沈黙」
```

- **α「解放」**: ナギサが自らの意志で消去を受容／超越する結末。視聴者の熱量が最大の時に到達。
- **β「残照」**: ナギサは消えるが、シンジの中に確かな痕跡を残す結末。記録の積み重ね（記録度）で到達。
- **γ「沈黙」**: 何もかもが"規定通り"に消える結末。視聴者が物語に十分関与しなかった場合の結末。

## 伏線管理の原則

- 1話に追加する伏線は **最大1件**。自然な流れを優先し、不要なら追加しない。
- 伏線 ID は `FS-XXX` の連番。`sheets/` の Foreshadowing シートで管理。
- 回収 Day は伏線追加時に `target_episode` として指定するが、前後にずれても問題ない（arc_plan 側は厳守、伏線は柔軟）。
