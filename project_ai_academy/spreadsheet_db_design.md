# Soul Reboot - スプレッドシートDB設計書

## 概要

このプロジェクトのすべての入出力データは **Google スプレッドシート** を簡易データベースとして管理する。
毎日の自律サイクルはこのスプレッドシートを読み書きし、人間（あなた）はここを見て修正・承認・追記ができる。

---

## スプレッドシート構成（シート一覧）

| シート名 | 役割 | 主な書き込みタイミング |
|---|---|---|
| `📋 Episodes` | 各話の概要・ステータス管理 | Architectが毎日書き込む |
| `📜 Scripts` | 台本（セリフ・ト書き）の全文 | Writerが毎日書き込む |
| `🔮 Foreshadowing` | 伏線の追加・回収トラッカー | Architectが管理 |
| `💬 Comments` | YouTube収集コメント・採用判定 | 毎日収集スクリプトが書き込む |
| `📊 Parameters` | 信頼度/覚醒度/記録度の3軸パラメータ推移 | 毎日自動更新 |
| `🎵 Assets` | 生成画像・音声・動画・BGMのパス/URL | 生成後に記録 |
| `📰 News` | 当日取り込んだリアルニュース | 毎日収集スクリプトが書き込む |
| `📖 Memory_L2` | 全話累積サマリー（JSON代替） | 各話完了後に書き込む |
| `⚙️ Config` | 全体のシステム設定値 | 手動で管理 |

---

## 各シートの詳細設計

### 📋 シート1: `Episodes`（エピソード管理台帳）

> AIの出力進捗を一目で確認・手動修正できる中枢シート

| 列 | カラム名 | 型 | 説明 | 例 |
|---|---|---|---|---|
| A | `話数` | 数値 | 話数 | 1 |
| B | `公開日` | 日付 | 公開日（現実の日付） | 2028-04-07 |
| C | `物語内の日数` | 数値 | 物語内の日数 | 1 |
| D | `フェーズ` | テキスト | フェーズ名 | 幸福な誤認 |
| E | `タイトル案` | テキスト | タイトル（AI案） | 100日後に君を忘れる。 |
| F | `確定タイトル` | テキスト | **手動で承認・修正したタイトル** | (空欄=AI案を使用) |
| G | `この話の目的` | テキスト | この話で達成すること | 出会いと悲劇の予感 |
| H | `感情曲線` | テキスト | 感情曲線 | 30→60→90 |
| I | `プロット要約` | テキスト | 3幕プロット要約（AI生成） | … |
| J | `クリフハンガー` | テキスト | クリフハンガー | … |
| K | `ステータス` | テキスト | 進行状況 | `PLANNED/SCRIPTED/VOICED/EDITED/PUBLISHED` |
| L | `YouTube_URL` | URL | 公開後のYouTube URL | https://... |
| M | `メモ` | テキスト | **手動メモ欄（修正指示など）** | ここに赤字でナギサを泣かせて |

### 📜 シート2: `Scripts`（台本DB）

> Writerが生成した台本を1行1行管理。TTSに渡すデータの源泉。

| 列 | カラム名 | 型 | 説明 |
|---|---|---|---|
| A | `話数` | 数値 | 話数 |
| B | `シーン番号` | 数値 | シーン番号（例: 1, 2, 3） |
| C | `シーン名` | テキスト | シーン名（例: 入学式・昇降口） |
| D | `画像プロンプト` | テキスト | 画像生成AI用プロンプト（英語） |
| E | `話者` | テキスト | 話者（`NAGISA` / `SHINJI` / `NARRATOR` / `SYSTEM`） |
| F | `セリフ・地の文` | テキスト | セリフ・地の文の本文 |
| G | `感情トーン` | テキスト | 感情トーン（静か/叫び/悲しみ/毒舌/震え） |
| H | `音声キャラ` | テキスト | 使用TTS設定（Gemini-TTS Voice名など） |
| I | `音声ファイルパス` | テキスト | 生成後の音声ファイルパス/URL |
| J | `承認済` | チェックボックス | **手動承認フラグ（FALSE=要確認）** |
| K | `メモ` | テキスト | **手動修正メモ** |

---

### 🔮 シート3: `Foreshadowing`（伏線トラッカー）

> 100話の整合性を守るための核心シート。AIは毎話ここを参照して「回収すべき伏線」を確認する。

| 列 | カラム名 | 型 | 説明 |
|---|---|---|---|
| A | `伏線ID` | テキスト | 固有ID（例: `FS-001`） |
| B | `追加話数` | 数値 | 伏線を張った話数 |
| C | `伏線内容` | テキスト | 伏線の内容 |
| D | `回収予定話数` | 数値 | 回収予定の話数（AI案） |
| E | `確定回収話数` | 数値 | **手動で修正した回収予定話** |
| F | `ステータス` | テキスト | `OPEN` / `RESOLVED` / `CANCELLED` |
| G | `回収済話数` | 数値 | 実際に回収した話数 |
| H | `回収メモ` | テキスト | どう回収したか |
| I | `重要度` | テキスト | 重要度（`HIGH` / `MID` / `LOW`） |

---

> YouTubeコメントを収集・フィルタリングし、物語に採用するかどうかを管理する。

| 列 | カラム名 | 型 | 説明 |
|---|---|---|---|
| A | `収集日` | 日付 | 収集日 |
| B | `対象話数` | 数値 | 対象話数 |
| C | `コメントID` | テキスト | YouTubeコメントID |
| D | `投稿者名` | テキスト | コメント投稿者名 |
| E | `コメント本文` | テキスト | コメント本文 |
| F | `いいね数` | 数値 | いいね数 |
| G | `AI感情分析` | テキスト | AI感情分析（応援/批判/考察/ネタバレ防止） |
| H | `採用スコア` | 数値 | AI採用スコア（0〜100） |
| I | `採用ステータス` | テキスト | `PENDING` / `ADOPTED` / `REJECTED` |

| 列 | カラム名 | 型 | 説明 |
|---|---|---|---|
| A | `話数` | 数値 | 話数 |
| B | `信頼度` | 数値 | 信頼度（0〜100） |
| C | `覚醒度` | 数値 | 覚醒度（0〜100） |
| D | `記録度` | 数値 | 記録度（0〜100） |
| E | `信頼度変化` | 数値 | 前話比変化量 |
| F | `覚醒度変化` | 数値 | 前話比変化量 |
| G | `記録度変化` | 数値 | 前話比変化量 |
| H | `変動トリガー` | テキスト | パラメータ変動のトリガー |
| I | `分岐フラグ` | テキスト | 分岐フラグ（発動した場合） |
| J | `手動調整` | チェックボックス | **手動で調整したか** |
| K | `メモ` | テキスト | **手動調整の理由メモ** |

| 列 | カラム名 | 型 | 説明 |
|---|---|---|---|

| C | `要約` | テキスト | 200字以内の要約 |
| D | `未回収の伏線` | テキスト | 未回収の伏線（カンマ区切り） |

---

### ⚙️ シート9: `Config`（システム設定）

> エンジンの動作を制御するマスターコンフィグ。ここを変えるだけで挙動が変わる。

| キー | 値 | 説明 |
|---|---|---|
| `設定キー` | 設定値 | 説明 |
| `CURRENT_EPISODE` | 1 | 今日生成する話数 |
| `PHASE` | `PHASE_1` | 現在のフェーズ |
| `AUTO_PUBLISH` | `FALSE` | 自動YouTube投稿（手動承認まで行わない） |
| `COMMENT_FETCH_COUNT` | 50 | 毎日収集するコメント数 |
| `NEWS_FETCH_COUNT` | 5 | 毎日収集するニュース数 |
| `GEMINI_MODEL` | `gemini-2.0-flash` | 使用するGeminiモデル |
| `PARAMETER_UPDATE_MODE` | `AUTO` | `AUTO` or `MANUAL` |
| `REAL_DATE_OFFSET_DAYS` | 0 | 現実日付と物語日付のオフセット |
| `SPREADSHEET_ID` | `(要設定)` | Google Spreadsheet ID |

## データフロー図

```
[毎日の自律サイクル]

  ① Config シートを読み込む
        ↓
  ② News シートに今日のリアルニュースを追加
        ↓
  ③ Comments シートに最新コメントを追加 + AI採用スコアリング
        ↓
  ④ Memory_L2 + L1 (直近3話) を読み込んでArchitectに渡す
        ↓
  ⑤ Architect がプロットを生成 → Episodes シートに書き込む
        ↓
  ⑥ Writer がプロット + ニュース + 採用コメントを受け取り台本生成
        → Scripts シートに書き込む（approved=FALSE で保存）
        ↓
  ⑦ Foreshadowing シートを更新（今話追加分 + 今話回収分）
        ↓
  ⑧ Parameters シートを更新（今話のパラメータ変動を記録）
        ↓
  ⑨ 〈承認待ち〉あなたがスプレッドシートを確認・修正・approved=TRUE に変更
        ↓
  ⑩ Assets 生成（画像/音声/BGMを生成し Assets シートに記録）
        ↓
  ⑪ 動画編集・YouTube投稿（AUTO_PUBLISH=TRUE の場合のみ自動）
        ↓
  ⑫ Memory_L2 + L1 を更新して翌日に備える
```

## 人間の介入ポイント（毎日の確認作業）

| タイミング | 確認シート | アクション |
|---|---|---|
| 朝（生成後） | `Episodes` | タイトル修正・notes欄に追加指示 |
| 朝（生成後） | `Scripts` | `approved`チェック・`notes`に修正指示 |
| 朝（生成後） | `Comments` | `manual_override`で採用可否の上書き |
| 朝（生成後） | `Assets` | 画像の`regenerate_flag`をTRUEにして再生成指示 |
| 随時 | `Foreshadowing` | 伏線の回収時期の調整 |
| 随時 | `Config` | システム全体の設定変更 |
